Request handling

Express passes the error.
Hono returns the response.

The DNS checks, redirects, parsing, and response contract are shared. The framework code only receives, validates, and returns the request.

Expressserver.ts
app.post('/api/inspect',
  async (req, res, next) => {
    try {
      const input = schema.parse(req.body)
      const result = await inspectUrl(
        input.url, 'express'
      )
      res.status(statusFor(result)).json(result)
    } catch (error) {
      next(error)
    }
  }
)
  • Node request and response objects
  • Middleware order is explicit
  • Errors move through next
Honoserver.ts
app.post(
  '/api/inspect',
  zValidator('json', schema),
  async (c) => {
    const input = c.req.valid('json')
    const result = await inspectUrl(
      input.url, 'hono'
    )
    return c.json(result, statusFor(result))
  }
)
  • Web-standard request model
  • Validated input is inferred
  • Handlers return a response
